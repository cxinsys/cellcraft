Preparing Tenet input files using Monocle2 Pseudotime

Abstract¶

 Single-cell gene expression studies offer the capability to profile the regulation of transcription in complex biological processes and heterogeneous cell populations. These studies are instrumental in discovering genes that characterize specific cell subtypes, capture intermediate states within biological processes, and differentiate between alternative cell fates. In these investigations, individual cells progress through gene expression programs independently, resulting in each cell representing a snapshot of the transcriptional program under investigation.


 There are several excellent tools for analyzing single-cell expression experiments. Among these tools, Monocle2 (see more details) stands out by introducing an innovative method for organizing individual cells based on pseudotime, positioning them along a path that mirrors a biological progression such as cell differentiation. This strategy capitalizes on the inherent asynchronous progression of these processes in individual cells. Monocle achieves this cell ordering by constructing a principled graph from the data obtained from single-cell genomics. Advanced machine learning techniques, including Reversed Graph Embedding, are employed to robustly and accurately unravel complex biological processes.

 Pseudotime is a metric representing the advancement of an individual cell within a process, like cell differentiation. In various biological processes, cells don't progress uniformly. In single-cell expression studies, particularly in processes such as cell differentiation, the captured cells can show a wide distribution of progress. This implies that among cells captured simultaneously, some might be at an advanced stage while others might not have initiated the process yet. This lack of sychrony poses challenges when trying to understand the sequence of regulatory shifts as cells transition between different states.


 So, our goal is to utilize the Pseudotime obtained from tools like Monocle to grasp the succession of regulatory changes that occur as cells transition between distinct states. For this goal, TENET enables the reconstruction of inter-cell type gene regulatory networks. The "TENET" approach uses a concept called transfer entropy (TE) to measure the amount of causal relationships between genes. This approach aims to predict large-scale gene regulatory cascades or relationships from scRNAseq data. According to the study, "TENET" outperforms other methods for reconstructing GRNs and is particularly successful in identifying key regulators from scRNAseq datasets. More Detailed principles and information are covered in the publication, so we encourage you to refer to it for understanding.

 The key data inputs from the single-cell expression dataset for TENET include "Pseudotime matrix", "Cell x Gene Matrix", and "Cell select file". If you follow this tutorial, you can obtain these 3 input files for GRN analysis using TENET

2-3. Ordering cells along the trajectory¶

Pseudotime serves as an abstract measure of progress, quantifying the distance between a cell and the trajectory's starting point along the shortest route. The trajectory's overall length is defined by the total transcriptional changes a cell undergoes while transitioning from the initial state to the final state.
The calculation of pseudotime is performed using the orderCells function, which can be time-consuming depending on the size of the data.
( The current data will take 1~2 minute)

In [14]:
cds_monocle_obj <- orderCells(cds_monocle_obj)

Following this, the plot_cell_trajectory function will be used to visualize the results.

In [15]:
# we can check Pseudotime along the cells
monocle::plot_cell_trajectory(cds_monocle_obj, color_by = "Pseudotime")

The results were also confirmed based on the annotation criteria.

In [16]:
# we can check seurat_annotation(celltype) on the pseudotime result
monocle::plot_cell_trajectory(cds_monocle_obj, color_by = "seurat_annotation")

"States" are defined based on branch points, separating clusters, and are considered to represent more similar biological processes than those not within the same state.

In [17]:
# we can check state on the pseudotime result
monocle::plot_cell_trajectory(cds_monocle_obj, color_by = "State")

4. Extracting the Cell x Gene matrix and Cell Select file¶

Retrun to Index

TENET takes a lot of runtime with every gene and cell. To bulid the accurate Gene Regulatory Network (GRN) and reduce the resource consumption, TENET recommends subsetting the gene set using DEG or highly variable genes from specific cell types. But we have already identified the DEGs in DEG analysis part, so we can easily obtain the Cell X Gene matrix using these.

# Make matrix using a DEG marker genes
pbmc_matrix <- as.data.frame(as.matrix(t(pbmc@assays[["RNA"]]@counts[unique(pbmc_subset.markers$gene),])))
